Built something interesting over the weekend.

A terminal-first ChatGPT wrapper that doesn’t just print responses — it works on structured JSON input/output.

Why?
Because I wanted more control.
And honestly… I didn’t want to burn tokens on APIs unnecessarily.

This setup lets you:
- Run ChatGPT-like workflows directly from terminal
- Use structured input/output (JSON) instead of plain text
- Plug it into pipelines, agents, or automation flows
- Treat it more like a programmable system than a chat UI

It’s not just a TUI.
It’s a step toward turning ChatGPT into a controllable agent layer without relying heavily on APIs.

This approach might not fully align with platform rules or intended usage patterns — so use it thoughtfully.
